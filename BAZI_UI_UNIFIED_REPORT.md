# 八字全域分析 UI 統一修正報告

版本：2026-06-26 / `bazi-suite.js 1.1.0`

## 修正目的

八字全域分析先前使用瀏覽器原生的 `input type="date"`、`input type="time"` 與國家／城市下拉選單，在 Samsung／Android 會跳出系統日曆，外觀與原版單盤不一致。

本次將全域分析中的出生資料輸入，統一為與原版單盤相同的黑金底部選擇器風格。

## 已統一的區域

- 單人分析
- 合盤 A 方
- 合盤 B 方
- 人格卡
- 真太陽時工具

## 已統一的元件

- 性別：黑金雙按鈕
- 國曆日期：站內日曆、年份與月份快速跳轉
- 出生時間：24 小時與分鐘十位／個位選擇
- 不知時辰：整合在時間選擇器內
- 子時換日口徑：黑金雙按鈕
- 出生地點：站內底部城市選擇器

地點資料仍使用原本 `solar-location.js` 的完整國家、城市、經緯度、固定時區備援與 IANA 時區資料，沒有退回只有台灣少數城市的簡化表。

## 行為修正

- 全域分析畫面不再生成任何 `input[type="date"]` 或 `input[type="time"]`。
- Android 不再呼叫系統原生日曆／時間視窗。
- 日期、時間、未知時辰、國家與城市仍寫回原有隱藏欄位，排盤核心與提示詞資料介面不變。
- 切換單人、合盤、人格、工具分頁時，已填資料會保存在原本的草稿狀態中。
- 城市選擇後仍使用該城市的經緯度、時區與 IANA timezone ID 計算真太陽時。
