# 八字模組修正報告（2026-06-25）

## 修正範圍

本次只處理八字排盤、真太陽時、歲運邊界、干支作用資料層、命理解讀模型分層及 AI 提示詞。未修改塔羅、紫微、梅花、占星等其他模組。

## 主要修正

1. **本地精確曆法底座**
   - 內嵌 `lunar-javascript 1.7.7`，不再依賴 CDN。
   - 四柱、節氣、立春年界與起運精確到秒。
   - 精確引擎缺席時才使用舊節氣表，並明確標示為近似，不冒充秒級精度。

2. **真太陽時與時區**
   - 改用 Meeus 均時差算法。
   - 支援 IANA 時區、歷史偏移與 DST。
   - DST 重疊時間採較早一次並標記；不存在時間採相容解析並警告。
   - 固定 UTC 偏移只作備援，提示詞明示限制。

3. **換日政策**
   - 前端可選 `23:00 子初換日` 或 `00:00 午夜換日`。
   - 政策寫入命盤資料與提示詞，避免流派口徑隱藏。

4. **精確起運與大運交界**
   - 修正原本算出月份後又只用整數歲數排運的問題。
   - 大運改用精確起運日期及半開區間 `[起點, 下一起點)`。
   - 現在時刻先轉成命盤採用的當地／真太陽牆鐘時間，再判現行大運，避免 UTC 時區錯位。

5. **刑沖合害資料層統一**
   - 由核心唯一計算三合、三會、半合、拱合、六合、六沖、六害、相破、相刑、自刑。
   - 移除提示詞與 upgrade 層的重複補算。
   - 六合、天干五合、三合三會只列配對／候選，不自動宣告合化。
   - 刑沖合害不再直接灌入五行權重或歲運吉凶分數。

6. **解讀模型降級與分層**
   - 五行比例、旺衰、喜忌、病藥、調候與歲運分數明示為本系統相對模型，不當成古籍固定百分比或客觀機率。
   - 從格、化氣格改列「待覆核候選」，不再由百分比門檻自動定格並覆蓋喜忌。
   - 調候與扶抑分開，不再因某五行偏低就自動插入用神。
   - 官殺明透與藏根分層，不再把藏殺直接敘述成暗中競爭等具體事件。
   - 暗合廣義算法預設停用，避免任意藏干五合造成大量假訊號。

7. **提示詞重構**
   - 分為「排盤事實層」與「流派模型層」。
   - 明示曆法引擎、精度、時區、真太陽時、換日政策、精確交運點及不確定性。
   - 禁止把合化、沖刑、神煞、分數直接寫成必然事件。
   - 未知時辰時，明確降低時柱、神煞與精確起運可信度。
   - 命理與水晶均加入證據限制，不宣稱保證改運、招財或治療。
