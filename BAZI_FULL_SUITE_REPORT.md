## 1. 範圍與誠實邊界

本套件重建的是 OpenFate 公開網站可觀察到的八字產品能力，以及公開 `bazi-mcp`／相關公開引擎所揭露的工具範圍，並整合進 `mytool-blue` 的既有前端架構。

本次沒有、也不宣稱取得或複製以下未公開內容：

- OpenFate 私有伺服器程式碼
- 私有合盤評分權重
- 私有 AI 系統提示詞
- BZTI 私有分類映射與命名規則
- 任何未公開資料庫或會員端服務

因此，本專案採用「公開功能等價重建」：排盤事實使用可檢查的本地曆法與規則；合盤、主題解讀及人格卡由本專案自行實作，程式碼可直接審查，不用假分數冒充原站私有算法。

## 2. 已完成的功能

### 2.1 精確單人八字排盤

- 公曆出生資料與男女命輸入
- 國家、城市、經度、IANA 時區
- 歷史 DST／夏令時間處理
- 真太陽時校正到秒
- `00:00 午夜換日`與`23:00 子初換日`政策
- 四柱、日主、十神、藏干、十二長生、納音、空亡
- 生肖
- 人元司令
- 原局天干五合與地支刑、沖、合、害、破、三合、三會、半合、拱合
- 配對存在與「是否成化」分離，不自動宣告合化
- 精確起運年月日時
- 大運採半開區間 `[起點, 下一起點)`
- 流年以立春為界
- 大運交界依所在地民用時間轉真太陽牆鐘後判定
- 未知時辰模式與可信度降級
- 稱骨／八字重量列為末位民俗輔助，不凌駕四柱生剋

### 2.2 六種單盤入口

1. 純排盤
2. 綜合命盤
3. 事業方向
4. 財富策略
5. 感情婚姻
6. 流年趨勢

所有提示詞都拆成：

- 排盤事實層：可重算核對
- 流派模型層：旺衰、扶抑、調候、格局、病藥等傳統判斷
- 不確定性與替代判法
- 精確大運、立春流年界線
- 禁止把命理推論寫成確定事件、金額、機率、疾病或法律結論

### 2.3 八種雙人情境合盤

1. 婚戀／婚姻
2. 事業合夥
3. 婆媳關係
4. 閨蜜／摯友
5. 父子關係
6. 母子關係
7. 一般朋友
8. 主管與部屬

合盤不是只把兩張命盤並排，而是包含：

- A 看 B、B 看 A 的雙向十神映射
- 對方四柱天干及藏干在自身日主下的十神
- 雙方日主五行生剋關係
- 雙方日柱與日支／夫妻宮互動
- 跨盤天干五合、生剋
- 跨盤地支六沖、六合、六害、六破、三刑、自刑、同支
- 需要雙方共同補足的三合、三會條件
- 五行互補與共同負荷；僅列模型證據，不生成虛假配對百分比
- 雙方當前大運與立春流年同步、錯峰及換運交界
- 依婚姻、合夥、親子、朋友、職場等角色切換分析問題與風險限制
- 任一方未知時辰時，排除該方時柱跨盤訊號並明確降級

### 2.4 本地 32 型人格卡

新增「靜月五軸人格」：

- 5 個可追溯軸向
- 32 個固定、可重現類型
- 同一命盤輸入會得到同一結果
- 可生成圖片分享卡
- 僅作文化型自我觀察，不作心理診斷或人格定論

此功能為本專案原創規則，不冒充 OpenFate 的 BZTI 私有算法。

### 2.5 曆法工具

- 真太陽時計算器
- 顯示民用時間、真太陽時、經度修正、均時差、DST、時區政策
- 四柱反查公曆候選時間
- 反查限制年份範圍，避免瀏覽器失控
- 反查結果明示為「民用鐘錶候選」，仍須帶出生地、時區與真太陽時複核
- 排盤政策說明卡

### 2.6 前端與資料功能

- 手機優先全螢幕介面
- 單人、合盤、人格、工具、紀錄五分頁
- 本機歷史紀錄，最多 30 筆
- TXT 提示詞下載
- JSON 完整資料下載
- 列印／另存 PDF
- 單盤、合盤、人格圖片分享卡
- 10 個 AI 入口：ChatGPT、Gemini、Claude、Grok、DeepSeek、Kimi、豆包、Meta AI、Copilot、Perplexity
- 保留舊版八字介面回退入口

## 3. 重要修正

- 修正舊程式起運雖算出月數、後續卻只取整歲的錯誤。
- 修正大運只按年份切換，改為精確日期時間。
- 修正 UTC 真實瞬間與命盤真太陽牆鐘比較基準錯位。
- 修正六害只在提示詞層補算、核心資料缺漏的問題。
- 刑沖合害改由核心唯一輸出，移除各提示詞重複計算。
- 六沖、刑害破只列觸發，不再套用「沖忌必吉／沖喜必凶」自動加減分。
- 六合、五合、三合三會不再自動宣告成化。
- 從格、化氣格只列待審候選，不以百分比直接定格並覆蓋喜忌。
- 調候與扶抑分離，不因某五行比例低便自動升為用神。
- 修正 standalone 八字無法取得精確農曆而導致稱骨資料缺失。
- 海外城市改用 IANA 時區，支援歷史 DST；固定 UTC 偏移只作明示限制的備援。
- 分享卡新增合盤及人格專屬圖卡。

## 6. 安裝方式

ZIP 內保留：

```text
mytool-blue-master/
```

將其解壓到原專案上一層，選擇覆蓋同名檔案即可。不要只把所有檔案丟進同一個資料夾，必須保留 `JS/`、`JS/vendor/`、`tests/` 的相對路徑。

GitHub 網頁版可逐一上傳；較穩定的方式是用 GitHub Desktop、Git 指令或手機解壓後整批覆蓋再提交。

## 8. 使用限制

- 命理屬傳統詮釋體系，不是科學預測。
- 合盤不等於婚姻、合作或親子關係的客觀評鑑。
- 不應用單一分數判斷適合、不適合、必婚、必離、背叛或成功。
- 財務、醫療、法律及心理問題仍需依實際資料與專業意見處理。
- 若出生時間不確定，時柱、精確起運、部分歲運同步及跨盤關係的可信度必須下降。
